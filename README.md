1. 전체 구현 내용

Spring Boot와 JWT를 활용해서 간단한 은행 시스템을 구현하였다.
회원가입, 로그인, 계좌 생성, 입금, 송금, 거래내역 조회 기능까지 기본적인 흐름은 구현하였다.
다만 보안 설정과 관리자 기능은 일부 미완성 상태이다.

2. 회원가입 및 로그인

회원가입 시 비밀번호는 BCrypt를 사용하여 암호화하였다.
로그인 시 아이디와 비밀번호를 비교하여 인증을 진행하고, 인증이 성공하면 JWT 토큰을 발급하도록 구현하였다.

또한 JwtAuthenticationFilter를 통해 요청이 들어올 때마다 토큰을 검사하고, 정상적인 경우 사용자 정보를 SecurityContext에 저장하도록 하였다.

3. 계좌 생성

회원가입이 완료되면 기본 계좌가 자동으로 생성되도록 하였다.
계좌번호는 UUID를 이용하여 생성하였고, 초기 잔액은 0원으로 설정하였다.

4. 입금 기능

입금 시 계좌를 조회한 후 금액이 0 이하인지 확인하고, 계좌가 동결 상태인지 검사하였다.
문제가 없을 경우 계좌 잔액을 증가시키고, 거래 내역을 저장하도록 구현하였다.

5. 송금 기능

송금 기능은 보내는 계좌와 받는 계좌를 각각 조회한 후 진행하도록 구현하였다.
금액이 0 이하인지, 잔액이 충분한지, 계좌가 동결 상태인지 등을 검사하였다.

또한 출금과 입금을 하나의 과정으로 처리하기 위해 @Transactional을 사용하였다.
잔액이 부족할 경우 RuntimeException을 발생시키고 전체 작업이 취소되도록 하였다.

6. 거래 내역 조회

거래 내역은 최근 5건만 조회하도록 구현하였다.
Spring Data JPA의 메서드 이름 규칙을 사용하여 최신 순으로 정렬된 데이터 중 상위 5개를 가져오도록 하였다.

입금과 송금 내역을 모두 조회할 수 있도록 senderAccount와 receiverAccount를 기준으로 조회하도록 구성하였다.

7. 보안 및 권한

JWT 기반 인증은 구현하였지만, 역할별 접근 제어는 완전히 적용하지 못하였다.
현재는 로그인 여부 중심으로 기능이 동작하는 상태이며, 관리자 권한 기능은 구현하지 못하였다.
